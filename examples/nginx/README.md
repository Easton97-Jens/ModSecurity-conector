# NGINX ModSecurity Examples

These examples show the production shape for request-only and bounded Phase 4
response-body inspection.

- Enable the connector with `load_module modules/ngx_http_modsecurity_module.so`,
  `modsecurity on`, and `modsecurity_rules_file`.
- Put base ModSecurity config under `/etc/modsecurity` and CRS under
  `/etc/modsecurity/crs`.
- Request-only mode sets `SecResponseBodyAccess Off`; it avoids late response
  disruption and is the conservative production default.
- Phase 4 mode sets `SecResponseBodyAccess On`, MIME restrictions, and a bounded
  `SecResponseBodyLimit`.
- `modsecurity_phase4_mode strict` records late interventions and can abort a
  connection after headers/body are committed. That is strict-abort evidence,
  not a clean HTTP 403.
- Connector Phase 4 decisions are JSON lines in
  `/var/log/modsecurity/nginx-phase4.jsonl`; access/error logs remain under
  `/var/log/nginx`.
- Audit logs are generated by libmodsecurity through `SecAuditLog`; rotate them
  with normal system log rotation.
- Keep compression disabled or ordered after inspection until your deployment
  proves whether inspected bytes are compressed or uncompressed.
- RESPONSE_BODY support remains bounded/non-promoted unless full-body behavior is
  separately proven in live runtime evidence.
