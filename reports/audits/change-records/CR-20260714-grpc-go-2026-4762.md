## Motivation and problem statement

<code>govulncheck</code> confirmed reachable advisory <code>GO-2026-4762</code>, an authorization bypass in gRPC-Go caused by a missing leading slash in <code>:path</code>. The original path was <code>cmd/msconnector-envoy-ext-proc/main.go:112:34: msconnector.main calls grpc.Server.Serve</code>. The module selected <code>google.golang.org/grpc v1.78.0</code>; the smallest reported fixed version is <code>v1.79.3</code>.

## Affected components and security boundaries

Only <code>connectors/envoy/ext_proc</code>'s Go module graph changed. The affected boundary is the ext_proc gRPC server accepting Envoy processor traffic. The module remains a Go 1.24 module; no <code>toolchain</code>, <code>replace</code>, or <code>exclude</code> directive changed. Framework files and other connector modules were not changed.

## Security impact

The reachable gRPC authorization-bypass advisory <code>GO-2026-4762</code> is closed for this module's checked dependency graph: the fixed gRPC version is selected and its previously reported <code>grpc.Server.Serve</code> path is no longer emitted by <code>govulncheck</code>. No application authorization logic, Envoy configuration, protocol behavior, logging, or defaults changed.

## Known limitations

- The native target did not run Common/libmodsecurity bridge tests because <code>MODSECURITY_INCLUDE_DIR</code> and <code>MODSECURITY_LIB_DIR</code> or <code>MODSECURITY_LIB_FILE</code> were not supplied.
- No H1, H2, H3, CRS, MRTS, or production runtime test was run; this update does not change their documented behavior.
- <code>staticcheck ./...</code> remains non-zero because of nine existing source findings: one <code>U1000</code> in <code>config.go</code>, seven <code>ST1005</code> findings in <code>processor.go</code>, and one <code>U1000</code> in <code>processor.go</code>.

They require a separately authorized Go toolchain update and were not changed here.

## Checks not run and rationale

- H1/H2/H3 and CRS/MRTS runtime matrix runs were not run because this isolated dependency update does not modify the Envoy service behavior and no prepared runtime environment was supplied.
- No tool binary was updated during the security evidence chain. The observed tools were Go <code>1.26.0</code>, Staticcheck <code>2026.1 (v0.7.0)</code>, and govulncheck <code>v1.6.0</code>.
