# Archived reports

**Language:** English | [Deutsch](README.de.md)

This directory retains superseded planning, readiness, and analysis reports.
Each retained report begins with a historical-status banner and preserves its
original dates, run IDs, and commit references. It is not current runtime
evidence.

## Retained generated snapshots

When a generated report has no newer verified input, its generator preserves
the prior evidence-bearing snapshot instead of replacing it with an empty
placeholder. The generated Markdown then states its current refresh status and
reason directly below its metadata. Read the current
[refresh manifest](../testing/generated/manifest/report-refresh-manifest.generated.md)
before treating a retained snapshot as a current report.

This rule preserves provenance without promoting an old full-matrix snapshot
as a result of the current compact six-connector core run.
