## Request-Body Evidence Added After This Run

The workflow and Make targets now include a targeted request-body smoke for
Envoy, Traefik, and Lighttpd:

- `make smoke-envoy-request-body`
- `make smoke-traefik-request-body`
- `make smoke-lighttpd-request-body`
- `make smoke-open-connectors-request-body`

Expected request-body evidence files:

- `$<CONNECTOR>_RESULT_ROOT/request-body-result.json`
- `$<CONNECTOR>_LOG_ROOT/request-body-decision.log`
- `$<CONNECTOR>_LOG_ROOT/request-body-request-transcript.jsonl`

The smoke uses `common/rules/modsecurity_request_body_smoke.conf`, POST bodies,
and rule `1000002`. It may set `request_body_smoke_verified=true` only when the
allowed POST returns 200 and the blocked body marker
`modsec-request-body-block` returns 403 from libmodsecurity. This is not
response-body verification.
